## [Unreleased]

### Added

- Add the IBKR Layer-1 to Layer-2 transaction normalizer (SP3): the broker-to-Portfolio-Performance export bridge stage that reads the per-institution Layer-1 rows (`transactions_interactive_brokers`) and writes broker-agnostic, classified, deduped rows to the previously orphaned Layer-2 table (`transactions_consolidated`), satisfying the `v_transactions_for_pp_export` gate. Adds the pure mapping function `normalize_ibkr_row` and the idempotent `TransactionNormalizer` persistence service (`storage/transaction_normalizer.py`), the canonical broker-agnostic `transaction_type` vocabulary (`BUY`, `SELL`, `DIV`, `DEPOSIT`, `WITHDRAWAL`, `TRANSFER_IN`, `TRANSFER_OUT`, typed as a `Literal` and asserted against the export-view CASE keys), the `account_mappings` table (broker `account_number` to PP `pp_group`/`pp_account`) with the `2a08a665e4e1` Alembic migration and the `uq_consolidated_source` Layer-2 idempotency constraint, security resolution by `isin` then `symbol`, and the `normalize-transactions` CLI command. Layer-2 `quantity` is stored as a positive magnitude with direction carried by `transaction_type` so the holdings view reproduces the signed Layer-1 quantity; net shares reconstructed from Layer-2 match SP2 on both a net-zero (cash merger) and a non-zero (TRADE + ACATS) anchor. Per-row persistence is fault-isolated via SAVEPOINTs (one failing row records to `NormalizationSummary.errors` rather than aborting the batch), and unrecognized broker subtypes are recorded in `NormalizationSummary.unknown_subtype_ids` for audit. Corporate-action `net_amount` follows `proceeds` when the Layer-1 `amount` is zeroed (the SP1 cashless fallback) so a cash merger does not lose its cash side
- Add IBKR position reconciliation (SP2): a read-mostly diagnostic that confirms the Layer-1 transactions ingested by SP1 reconstruct the holdings the broker reports, and flags drift. Parses and idempotently persists IBKR `<OpenPosition>` snapshots into the new `ibkr_open_positions` table (`extractor/ibkr_positions.py`; `storage/position_models.py` with the abstract `PositionSnapshotBase`, a per-provider sibling of `transaction_models`), reconstructs net share quantity per security from Layer-1 share-moving records (`TRADE`/`CORP_ACTION`/`TRANSFER`, bounded by `transaction_date <= report_date`, grouped by `COALESCE(isin, conid)`; CASH does not move shares), and reports per-security drift via the `reconcile-positions` CLI command and the `v_ibkr_position_reconciliation` PostgreSQL view, with the MATCHED/DRIFT/RECONSTRUCTED_ONLY/REPORTED_ONLY status taxonomy and a configurable `--tolerance` (default `0.0001` shares). Adds the `32b7a96122d0` Alembic migration (table, indexes, the NOT-NULL `(account_number, report_date, conid)` idempotency key, and the reporting view) and 17 tests, including the DBJA cash-merger-nets-to-zero and BMBCX-drift acceptance anchors. The Python `reconcile_positions()` path is authoritative; the view is the PostgreSQL reporting surface kept aligned with it
- Add Tier-3 listed-equity classification and the minimal external-API framework (ADR-003 Tier 3, ADR-005). New `security_master/external/` package: typed `ExternalAPISettings` (pydantic-settings, read from `.env`, with an https-URL guard, bound checks, and a `cache_path` validator enforcing the ADR-015 licensing invariant in code), an on-disk SQLite `ResponseCache` (TTL, provider-namespaced, degrades to a miss on SQLite errors), a synchronous `ExternalHTTPClient` base (tenacity retry/backoff on 429/5xx and transport errors, per-provider rate limiting, cache read-through, and graceful `ExternalAPIError` on exhausted retries or non-JSON/non-2xx responses), and `OpenFIGIClient` (identity + instrument-type) and `SECEdgarClient` (symbol to CIK to zero-padded SIC) provider clients. `classifier/equity.py` implements `classify_equity` (confirms equity via OpenFIGI, resolves a GICS sector from the provider sector or a SEC EDGAR SIC fallback, writes provenance, honors the override lock, logs and degrades to unresolved on API failure), and `classifier/taxonomy_lookup.py` adds `resolve_gics_sector_by_code`. Re-statuses ADR-005 and ADR-003 to Partially Accepted
- Add IBKR Layer-1 non-trade transaction coverage (SP1): parse and idempotently persist `<CashTransaction>`, `<CorporateAction>`, and `<Transfer>` records into `transactions_interactive_brokers`, discriminated by a new `record_type` column (`TRADE`/`CASH`/`CORP_ACTION`/`TRANSFER`) and deduplicated on the universal `transactionID`. Adds `extractor/ibkr_flex_records.py` (the three element parsers and frozen dataclasses), the shared `extractor/_flex_common.py` tolerant date/decimal/string helpers, nullable non-trade tail columns (`action_id`, `action_description`, `direction`, `dividend_type`, `ex_date`, `proceeds`, `realized_pnl`, `figi`, `conid`) with the additive `71ab9a687378` Alembic migration, and per-record-type counts in the `import-broker` CLI. Corporate-action `amount` rounds to cents while `proceeds`/`realized_pnl` keep raw `Numeric(18, 6)` precision (RAD-tagged); absent amount/currency fall back to `Decimal(0)`/`USD` for cashless rows, documented with `#ASSUME`/`#VERIFY` markers
- Add the Entity Registry (ADR-016 Phase E1): `storage/entity.py` with the `Client` and `LegalEntity` ORM models, the `entity_type -> tax_form` map (`ENTITY_TYPE_TAX_FORMS`, five COA forms 1040/1065/1120/990/1041) and `default_tax_form_for()`, cross-system identity columns (`xero_organisation_id`, `xero_crypto_client_id`, `pp_portfolio_ref`), and the `d90f75a3b03e` Alembic migration creating the `clients` and `legal_entities` tables
- Add the IBOR to Xero GL crosswalk (ADR-016 Phase E2): `taxonomies/accounting-xero-gl.taxonomy.json` ("Accounting (Xero GL)" PP taxonomy keyed by 8-digit Xero account code) and `crosswalks/ibor_to_xero_gl.yaml` (classification to GL code mapping with `REVIEW` markers and an `unresolved` list for cash codes pending the master COA), guarded by a crosswalk-to-taxonomy consistency test
- Add the PP XML round-trip importer: `patch/pp_xml_import.py` with `parse_client()` (DB-free, unit-testable parser) and `PPXMLImportService` (idempotent persistence by ISIN, account/portfolio uuid, and bookmark label+pattern); the `pp-master` Click CLI with `import-xml` / `export-xml` console entry point; and the Alembic baseline migration (`sql/versions/`, 21 tables) with `alembic.ini` and `sql/env.py`
- Add unique constraints backing importer idempotency: `pp_client_config.config_name` (`uq_pp_client_config_name`) and `pp_bookmarks (label, pattern)` (`uq_pp_bookmark_label_pattern`)
- Add `.pre-commit-config.yaml` with SHA-pinned remote hooks (ruff v0.15.12, detect-secrets v1.5.0, commitizen v4.13.10, yamllint v1.38.0, markdownlint-cli v0.48.0) and local `poetry run` hooks (basedpyright, bandit, darglint, interrogate, no-em-dash pygrep); add `pre_commit` nox session that installs and runs hooks

- Add `qlty` CLI integration with `.qlty/qlty.toml` configuration covering bandit, ruff, and basedpyright plugins
- Expand Ruff rule set to PyStrict-aligned standard: `ANN`, `ARG`, `ASYNC`, `C4`, `DTZ`, `ERA`, `FBT`, `FLY`, `FURB`, `G`, `ICN`, `INT`, `ISC`, `LOG`, `PERF`, `PGH`, `PIE`, `PL`, `PT`, `PTH`, `PYI`, `Q`, `RET`, `RSE`, `RUF`, `S`, `SIM`, `SLOT`, `T10`, `T20`, `TC`, `TID`, `TRY`, `UP`, `W` rule groups added to `[tool.ruff.lint] select`

- Add `scripts/validate_taxonomy_json.py`, a Portfolio Performance taxonomy JSON validator enforcing the importer contract (required `name`, `#RRGGBB` colors, unique classification keys, non-empty instrument `identifiers`, and assignment `weight` bounds and per-instrument sum), wired as the `validate-taxonomies` local pre-commit hook scoped to `taxonomies/*.taxonomy.json`

- Add `renovate.json` extending `config:recommended` with `enabledManagers` set to `pep621`, `github-actions`, `pre-commit`, and `pip_requirements`
- Replace `mypy` with `basedpyright` in strict mode across `pyproject.toml`, `noxfile.py`, `Makefile`, and `.github/workflows/ci.yml`
- Remove `[tool.mypy]`, `[[tool.mypy.overrides]]`, and `[tool.pydantic-mypy]` config blocks from `pyproject.toml`; add `[tool.basedpyright]` block with `typeCheckingMode = "strict"`

- Tighten type annotations in `mappers.py` (bare `dict` and `list[dict]` to `dict[str, Any]` / `list[dict[str, Any]]`) and `validators.py` (`list[str]` annotations on local variables)
- Switch XML generation in `pp_xml_export.py` to use `defusedxml.ElementTree` for all runtime XML operations; use `TYPE_CHECKING` guard so BasedPyright reads stdlib stubs while the runtime import stays on defusedxml
- Replace `black` formatter with `ruff format` across `noxfile.py`, `Makefile`, and CI workflow
- Replace `safety` vulnerability scanner with `pip-audit` in `noxfile.py`, `Makefile`, and CI workflow

### Fixed

- Fix `pp_xml_export.py` building XML through `defusedxml.ElementTree`, which omits the writer API (`Element`/`SubElement`/`tostring`) and raised `AttributeError` at runtime; it now builds with stdlib `ElementTree` and parses untrusted input with `defusedxml`
- Widen `PPSecurityPrice.price_value` from `Integer` to `BigInteger`: PP stores prices as value * 1e8, which overflows a 32-bit integer above ~$21
- Set `viewonly=True` on the two `PPTransactionUnit` navigation relationships to resolve an overlapping-foreign-key mapper error on the soft-polymorphic `transaction_type` key
- Fix the PP XML round-trip integration test skipping only when `DATABASE_URL` is unset, which the autouse conftest fixture always sets; it now probes connectivity and skips when PostgreSQL is unreachable, so CI legs without a database service skip instead of erroring
- Add `# nosemgrep: python.lang.maintainability.return.return-not-in-function` to all SQLAlchemy `mapped_column` lambda defaults in `models.py`, `pp_models.py`, and `transaction_models.py`; semgrep misreads the implicit lambda return as a bare return-not-in-function
- Broaden `except` clause in `PPXMLExportService.validate_export` to catch `defusedxml.DefusedXmlException` in addition to `ET.ParseError`; defusedxml security violations do not inherit from `ParseError`
- Add `try/except defusedxml.DefusedXmlException` around `defused_minidom.parseString()` in `_prettify_xml` to convert XML security violations to `ValueError` instead of propagating unhandled
- Correct `qlty.toml` format and add path exclusions for qlty 0.612.0 compatibility
